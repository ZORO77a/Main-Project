Step 2: Encryption Process

The file goes through hybrid encryption using CRYSTALS-Kyber768 and AES-256-GCM:

Algorithm Chain:
1. Generate Kyber768 ephemeral keypair
2. Encapsulate shared secret using public key
3. Derive AES-256 key via HKDF-SHA256
4. Encrypt file with AES-256-GCM
5. Bundle: [public_key][kyber_ciphertext][iv][tag][encrypted_data]
Component Size Purpose
Kyber Public Key 1,184 bytes For post-quantum key encapsulation
Kyber Ciphertext 1,088 bytes Encapsulated shared secret
AES IV (Nonce) 12 bytes Initialization vector for AES-GCM
AES GCM Tag 16 bytes Authentication tag for integrity
Encrypted File Data Variable Your actual encrypted content

Total Size: ~2,300+ bytes (depends on file size)
